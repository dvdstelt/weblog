This could not have arrived at a better time! I just heard from [Marc van de Wert](http://www.marcvandewert.nl/PermaLink,guid,41501171-af46-447c-9637-0bad46530b5c.aspx) that [Windows Workflow Foundation Beta 1.2](http://www.marcvandewert.nl/ct.ashx?id=41501171-af46-447c-9637-0bad46530b5c&url=http%3a%2f%2fwww.microsoft.com%2fdownloads%2fdetails.aspx%3fFamilyID%3d589e23dd-503e-4179-9858-d080c669f8b9%26DisplayLang%3den) was just released. The good news is, that it can run on the RTM version of Visual Studio 2005, where the previous version ran (or actually still runs) only on VS2005 Beta 2.
At my current project, where [Marc](http://www.marcvandewert.nl/default.aspx) is also on my team, we just decided to use Windows Workflow Foundation (WF). [Anko](http://www.ankoduizer.nl/PermaLink,guid,2d4b8d29-c15b-4170-8101-76cd6dc58acd.aspx), who dove deep into WF since it went public on the PDC, convinced me to use it, after a short introduction at the [office](http://www.class-a.nl/). But because of the VS2005 Beta 2 restriction, we couldn’t actually connect to WF yet. Now we can! While a lot of people are talking about it, we’re probably one of the first to actually implement it in a real life project! From the technology part, this is going to be one of my most exciting projects ever, because we’re using a lot of other great (new) technologies.

Oh, be sure to install the [december Pre-release WinFX runtime components](http://www.microsoft.com/downloads/details.aspx?FamilyId=BD3BA2D5-6ADB-4FB2-A3AA-E16A9EA5603F&displaylang=en) first.
**update** : Go figure, I just wanted to download this version, it was already on my laptop. While downloading it a few days ago, I had no idea this version was the latest and compatible with my VS2005 RTM release… 🙂  
 Typical by the way, that I cannot find info on this release on any weblog. Aren’t people excited about WF?
